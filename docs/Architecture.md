## 核心模块

### 1. 工作流引擎 (Workflow Engine)

工作流引擎是系统的核心，负责：
- 解析工作流定义
- 调度任务执行
- 管理依赖关系
- 追踪执行状态

```
WorkflowEngine
├── parse()      # 解析工作流定义
├── validate()   # 验证工作流
├── execute()    # 执行工作流
├── pause()      # 暂停执行
├── resume()     # 恢复执行
└── stop()       # 停止执行
```

### 2. Agent 管理器 (Agent Manager)

Agent 管理器负责所有 Agent 的生命周期：
- Agent 注册/注销
- 资源分配
- 状态监控
- 消息路由

```
AgentManager
├── register()      # 注册新 Agent
├── unregister()    # 注销 Agent
├── get()           # 获取 Agent
├── list()          # 列出所有 Agent
├── allocate()      # 分配资源
└── monitor()       # 监控状态
```

### 3. 模型路由器 (Model Router)

模型路由器提供统一的 LLM 调用接口：
- 模型选择
- 请求路由
- 负载均衡
- 错误重试

```
ModelRouter
├── call()          # 调用模型
├── stream()        # 流式调用
├── getEmbeddings() # 获取嵌入
└── switchModel()   # 切换模型
```

### 4. 任务调度器 (Scheduler)

调度器负责任务队列和优先级管理：
- 队列管理
- 优先级排序
- 并发控制
- 失败重试

```
Scheduler
├── schedule()      # 调度任务
├── process()       # 处理队列
├── getStats()      # 获取统计
├── pause()         # 暂停调度
└── resume()        # 恢复调度
```

## 数据流程

### 工作流执行流程

1. **创建工作流**: 用户通过前端拖拽节点
2. **保存配置**: 工作流保存到数据库/本地存储
3. **解析工作流**: 引擎解析节点和连接
4. **拓扑排序**: 确定执行顺序
5. **调度任务**: 将任务加入队列
6. **执行 Agent**: 按顺序调用 Agent
7. **收集结果**: 聚合输出结果
8. **更新状态**: 实时更新前端界面

### Agent 间通信

```
User Input
    ↓
Planner Agent (分解任务)
    ↓
[Researcher Agent, Designer Agent] (并行)
    ↓
Coder Agent (代码生成)
    ↓
Tester Agent (测试)
    ↓
Reviewer Agent (质量检查)
    ↓
Final Output
```

## 技术栈

### 前端
- **框架**: Vanilla JavaScript (无需框架)
- **构建工具**: Vite
- **样式**: CSS + CSS Variables
- **字体**: Orbitron, Rajdhani

### 后端
- **框架**: Express.js
- **实时通信**: Socket.IO
- **任务队列**: BullMQ
- **日志**: Winston
- **验证**: Zod

### 集成
- **OpenAI API**: GPT 模型
- **Anthropic API**: Claude 模型
- **Ollama**: 本地模型

## 扩展性设计

### 插件系统
系统支持插件扩展：
- 自定义 Agent 类型
- 自定义模型集成
- 自定义节点类型
- 自定义输出格式

### 配置系统
支持多环境配置：
- Development
- Staging
- Production

### 监控系统
- Prometheus metrics
- 健康检查端点
- 告警规则配置

## 性能优化

- **连接池**: 复用数据库/API 连接
- **缓存**: 缓存频繁访问的数据
- **批处理**: 批量处理相关任务
- **流式响应**: 使用 SSE 减少延迟
