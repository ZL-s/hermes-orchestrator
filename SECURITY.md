# 安全策略

## 报告安全问题

如果您发现安全漏洞，请不要通过公开的 GitHub Issues 报告，而是通过以下方式联系我们：

- 邮件: security@example.com
- GitHub: 通过私人仓库或 Discord 私信

我们会在 48 小时内回复，并在确认问题后尽快发布修复。

## 支持的版本

| 版本 | 支持状态 |
|------|----------|
| 1.0.x | ✅ 完全支持 |
| < 1.0 | ❌ 不支持 |

## 安全最佳实践

使用本项目时，请遵循以下安全准则：

### 1. API 密钥管理
- 永远不要提交 API 密钥到代码仓库
- 使用环境变量存储密钥
- 定期轮换密钥

### 2. 网络安全
- 在生产环境使用 HTTPS
- 配置 CORS 白名单
- 启用速率限制

### 3. 输入验证
- 验证所有用户输入
- 净化 LLM 提示词
- 限制请求大小

### 4. 日志安全
- 不要记录敏感信息
- 使用结构化日志
- 定期审查日志

## 漏洞披露流程

1. **报告**: 通过安全邮件报告问题
2. **确认**: 我们确认收到并开始调查
3. **修复**: 开发并测试修复方案
4. **发布**: 发布安全更新
5. **公告**: 发布安全公告
